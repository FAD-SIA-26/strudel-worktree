# Strudel ORC Reference

- Default style bias: synthwave first, unless the user goal explicitly names another supported genre.
- `data/strudel-examples` is the primary creative source. Stay as close as possible to those examples before inventing new structures.
- For synthwave runs, prefer `data/strudel-examples/synthwave/*` first and only fall back to other genres when the target pattern is missing.
- Reuse as much as possible from the examples: instrumentation, arp shape, chord movement, bass motion, drum phrasing, effect choices, and arrangement structure.
- Use the Strudel docs only to understand or correctly apply instruments/effects already suggested by the examples.
- Samples docs: https://strudel.cc/learn/samples/
- Synth docs: https://strudel.cc/learn/synths/
- Effects docs: https://strudel.cc/learn/effects/
- Preserve section ownership exactly: write only the file named in the task and keep the required export name unchanged.
- Prefer loopable parts, clear arrangement roles, and synthwave-friendly timbres such as saw, square, warm pads, and restrained tempo-synced effects.
